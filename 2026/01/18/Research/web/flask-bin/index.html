Flask / Werkzeug Debug PIN – Full Theory & Exploitation Guide

1. Tổng quan bài toán

Dạng bài này xuất hiện rất thường xuyên trong CTF Web / Pentest lab khi:

  • Server chạy Flask ở chế độ debug
  • Debugger của Werkzeug được bật
  • Truy cập debugger yêu cầu PIN
  • Ứng dụng tồn tại primitive đọc file tùy ý (LFI, path traversal, SSRF → file)

Mục tiêu cuối cùng:

Tính toán chính xác Werkzeug Debug PIN để truy cập interactive console → RCE

2. Điều kiện tiên quyết (Prerequisites)

Bài bắt buộc phải có ít nhất một trong các primitive sau:

  • LFI: ../../../../etc/passwd
  • Read arbitrary file
  • SSRF đọc được file://
  • Debug endpoint leak source code

Nếu không đọc được file hệ thống, bài này không giải được.

5. Bước 2 – Lý thuyết Werkzeug Debug PIN

5.1 Công thức tổng quát

Werkzeug sinh PIN từ 2 nhóm dữ liệu:

Public bits (có thể đoán / đọc được)

  • username
  • modname
  • appname
  • path tới file Flask

Private bits (bắt buộc phải leak)

  • MAC address (uuid.getnode())
  • Machine ID (/etc/machine-id)
  • Cgroup suffix (/proc/self/cgroup)

10. Checklist giải bài nhanh

  • Có primitive đọc file
  • Xác định MD5 hay SHA1
  • Đọc /etc/machine-id
  • Đọc /proc/self/cgroup
  • Lấy MAC address
  • Xác định flask path
  • Generate PIN offline
  • Truy cập debugger → RCE

11. Lưu ý thực tế CTF

  • Docker luôn có cgroup
  • Sai 1 byte → PIN sai
  • MAC phải convert đúng integer
  • Sai appname/modname → brute

12. Tổng kết

Werkzeug Debug PIN KHÔNG phải random

Nó là:

  • Deterministic
  • Phụ thuộc hệ thống
  • Có thể tính offline nếu leak đủ thông tin

👉 Đây là design flaw có chủ đích, phục vụ debug, nhưng trở thành critical vuln trong CTF.
