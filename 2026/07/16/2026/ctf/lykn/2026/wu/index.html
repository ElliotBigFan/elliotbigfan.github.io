8. Giải thích các field quan trọng trong fake FILE

Offset 0x00: command string

Khi chain gọi system(fp), fp trỏ tới đầu FILE object. Vì vậy đầu object cũng chính là command string.

b' sh\x00' hợp lệ vì shell ignore leading space.

Nếu muốn in flag trực tiếp, có thể dùng:

Khi đó không cần interactive shell.

Offset 0x88: _lock

FILE object cần _lock trỏ tới vùng hợp lệ để không crash khi glibc lock/unlock stream.

Offset 0xA0: _wide_data

Trỏ _wide_data vào chính vùng stdout đã bị ta control.

Offset 0xC0: _mode

Set mode để stream đi theo wide-oriented path. Trong một số build để 0 vẫn có thể chạy, nhưng set -1 giúp ổn định hơn cho nhánh wide FILE.

Offset 0xD8: vtable hợp lệ

Glibc 2.39 có vtable validation. Nếu vtable trỏ lung tung sẽ bị abort. _IO_wfile_jumps nằm trong vùng vtable hợp lệ của libc, nên qua check.
