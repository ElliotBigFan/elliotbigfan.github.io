1. Tổng quan

Trong bài IPKage, backend dùng FastAPI và có một tính năng upload/restore cache xử lý tài liệu trademark. Cache này được lưu dưới dạng file .pkl, sau đó backend đọc lại bằng pickle.load().

Vấn đề nằm ở chỗ: pickle không phải format dữ liệu an toàn để nhận từ user. Khi Python unpickle một object, nó có thể gọi lại các callable được chỉ định trong payload. Nếu attacker kiểm soát file pickle, attacker có thể biến thao tác pickle.load() thành thực thi code trên server.

7. Root cause

Vulnerability nằm ở thiết kế:

Do pickle.load() có thể thực thi callable từ payload, attacker có admin token có thể biến chức năng restore cache thành RCE.

Fix đúng hướng:

  • Không dùng pickle cho dữ liệu user kiểm soát.
  • Dùng JSON hoặc format dữ liệu an toàn.
  • Nếu bắt buộc dùng pickle nội bộ, không cho user upload trực tiếp file pickle.
  • Ký/HMAC cache file và verify trước khi load.
  • Tách quyền upload cache khỏi quyền restore cache.
  • Dùng allowlist schema thay vì deserialize object tùy ý.
