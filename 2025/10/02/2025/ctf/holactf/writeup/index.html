Solution

1. Xác định vị trí xảy ra lỗi bảo mật trong mã nguồn

  • Vì challenge có cho tính năng upload file nên ta sẽ thử kiểm tra liệu có thể tận dụng điều này để upload trực tiếp webshell và khai thác hay không?
  • Ở file upload.php, có 2 dòng code là:

  • 2 dòng code trên sẽ đảm bảo ta chỉ được phép upload file có đuôi file cuối cùng thuộc $allowedExtensions, vậy nên ta không thể upload webshell có đuôi file .php để rce được. Thế nên ta sẽ nghĩ theo hướng khác.

  • Ở cả 3 file home.php, upload.php, view_avartars.php đều unserialize trực tiếp cookie của user.

  • Vậy ta đã xác định được nơi để truyền untrusted data vào là trường cookie.

2. Tìm các gadgets có sẵn trong mã nguồn để tạo thành exploit chain RCE

  • Ở cả 2 file upload.php và view_avatars.php, đểu require đến file là LogAndCheck.php, nhìn qua file này, ta có thể thấy được có 2 class giúp ta RCE là LogFile và Logger.

    • Với hàm fwrite ở class Logger, ta có thể nghĩ đến việc tận dụng gadget này để ghi webshell và rce. Tuy nhiên, dòng code để unserialize cookie unserialize(base64_decode($_COOKIE['user']), ['allowed_classes' => ['User', 'LogFile']]) chỉ cho phép chuỗi serialize chứa 2 class là User và Logfile, vậy nên ta sẽ không thể tận dụng trực tiếp class LogFile nữa mà sẽ phải trigger class này thông qua hàm md5_file ở class Logger.
    • Về cơ bản, md5_file($path), nó mở file (qua PHP stream wrapper nếu có, ví dụ phar://, http://, file://), đọc nội dung file và trả về MD5 hash của nội dung đó (kiểu string 32 ký tự hex). ví dụ:

    Với phar:// có tác dụng phụ quan trọng: khi PHP mở phar để truy xuất file trong archive, nó sẽ đọc metadata của phar và unserialize nó — điều này có thể kích hoạt __wakeup(), __destruct() hoặc các phương thức khác của object trong metadata.

    Note: Nếu metadata chứa object mà class đó chưa được định nghĩa, PHP sẽ tạo __PHP_Incomplete_Class và các magic không chạy như mong đợi. Trên PHP 8+, phar.require_hash = On có thể ngăn metadata được unserialize nếu archive không có signature hợp lệ.

3. Poc (Proof of Concept)

  • Đầu tiên cần 1 file để tạo payload.phar (chứa class Logger). Đặt tên make_phar.php
  • Chạy file make_phar.php với lệnh:

  • không upload payload.phar vì server chỉ chấp nhận .jpg, .gif, .png, .jpeg mà ta sẽ đổi file thành payload.gif vì phar:// nhận diện file phar thông qua nội dung bên trong file nên để payload.gif vẫn sẽ được phar:// coi là một file phar.

  • Upload payload.gif vừa tạo ra lên server.

  • Kế tiếp, ta cần tạo một chuỗi serialize được encode bằng base64 tạo thành gadget chain trigger được hàm md5_file tại phar://upload/<username>/payload.gif

  • Ta sẽ tạo 1 file đặt tên gen_serial.php, chạy lệnh: php gen_serial.php và copy đoạn base64 vừa được tạo ra để đưa vào cookie.

  • Kế tiếp truy cập http://localhost:5005/?page=view_avatars để trigger được magic method destruct của class LogFile(truy cập file view_avatars.php hoặc upload.php vì 2 file này require LogAndCheck.php nơi chứa class Logger(sink)).

  • Sau đó truy cập: http://localhost:5005/s.php?0=cat%20/*.txt thực thi webshell vừa up để lấy flag.
