Web | Safe upload file

1. Phân tích mã nguồn và ý tưởng

  • Đọc qua file .yar (validate file upload), ta có thể thấy các file khi đc upload lên server sẽ được file .yar này validate rất chặt và gần như không thể bypass để upload webshell được.
  • Tuy nhiên, khi phân tích file upload.php ta có thể thấy một số đoạn code xử lý không hợp lý và dẫn đến upload file race condition.

  • Webshell mà ta sẽ upload lên đầu tiên sẽ được server đưa vào thư mục $tmp_path rồi sau đó server sẽ sleep 0.8s rồi mới xóa webshell của ta.

  • Vậy nếu ta có thể truy cập đúng file webshell của ta trước khi bị nó bị xóa đi thì ta hoàn toàn có thể RCE và đọc được flag.

2. Các bước khai thác:

  • Đầu tiên ta cần biết webshell mà ta upload đã được đổi tên như thế nào. Dưới đây là đoạn code xử lý đường dẫn đến file được upload.

  • Vậy ta có thể xác định được ta cần truy cập http://localhost:8001/tmp/<random 0-9999>.php để có thể thực thi webshell.

  • Ta cần viết một script có thể tạo được nhiều luồng để có thể nhanh chóng truy cập được vào web shell trước khi nó bị xóa.

  • Tới đây thì mình cũng chưa biết phải tạo script đó như thế nào nên chuyển sang dùng tool và mình dùng burpsuite với extension được tích hợp là Turbo Intruder.

  • Trước tiên mình sẽ bắt gói tin POST upload file rồi gửi qua tab Repeater và sau đó gửi gói tin http GET /tmp/%s.php qua extension Turbo Intruder.

  • Ở mục dùng script python nào cho cuộc tấn công race condition này thì mình dùng script này: https://github.com/PortSwigger/turbo-intruder/blob/master/resources/examples/race-single-packet-attack.py.

  • Chỉnh sửa nội dung script trên một tí cho phù hợp với bài CTF này rồi paste vào turbo intruder:

  • Hành vi của đoạn code trên là dạng concurrent / multi-connection (tương đương “multi-threaded” ở mức khái niệm), khi queue nhiều request rồi cho gửi đồng loạt bằng openGate.
  • Ở đây thì mình chỉ cho gửi 1200 request thôi vì đã thử để 10000 nhiều lần nhưng bị chậm và bị xóa file mất. Vì như thế nên bài này mình sẽ cùng 1 lúc chạy turbo và gửi file nhiều lần cho đến khi tên file rơi vào khoảng từ 0000.php đến 1200.php thì sẽ ok.
  • Và sau 7749 lần thì flag cũng đã về tay.
